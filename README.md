# 🔐 セキュア検証環境：KVM上の独自SSL対応Webサーバー構築

LinuCおよびCCNAの学習の一環として、外部ネットワークから隔離されたローカル環境（KVM）内に、セキュリティを強固に保ったWebサーバー環境を自力で構築・検証したプロジェクトです。

## 🌐 1. システム構成・パラメーター
物理ホストPC（親玉PC）の内部に仮想ネットワークを構築し、NATの壁を越えて手元PCから安全にアクセスする環境を実装しました。

### 💻 環境仕様
| 項目 | 設定値 | 導入の狙い / 補足 |
| :--- | :--- | :--- |
| **物理ホスト（踏み台）** | Dell OptiPlex 7060 SFF | 内部仮想マシンへのアクセス中継地（隔離環境の維持） |
| **ゲストOS (VM)** | Rocky Linux 9 | RHEL準拠の安定したサーバーOSの選定 |
| **仮想マシンの固定IP** | 192.168.122.7 | 起動ごとのIP変動を防ぎ、運用の安定性を確保 |
| **Webサーバー** | Nginx | 軽量・高速なリバースプロキシ・Webサーバーとしての選定 |
| **通信プロトコル** | HTTPS (TCP 443) | 独自SSL証明書によるトランスポート層の暗号化 |

---

## 🛠️ 2. 実装した主要なセキュリティ（鉄壁化）

本環境では、実務の本番環境や社内システムを想定し、以下の3つの防壁を構築しています。

### ① SSHの鍵認証化（パスワードログインの完全禁止）
* rootユーザーへのパスワード総当たり攻撃（ブルートフォース攻撃）を完全に防ぐため、パスワード認証を拒否。
* 強固な暗号化アルゴリズム（Ed25519）を用いた公開鍵認証のみを許可し、秘密鍵を持つ手元PCからのみアクセス可能に制限。

### ② 防壁（Firewalld）によるアクセス最小化
* 仮想マシン内部の不要なポートをすべて遮断。
* `firewall-cmd` を用い、Web通信に必要なポートのみをピンポイントで開放。

### ③ OpenSSLによる通信の暗号化（独自SSL/TLS化）
* `OpenSSL` コマンドを使用し、2048bitの秘密鍵と自己署名電子証明書（オレオレ証明書）を自作。
* Nginxの設定ファイル（`ssl.conf`）を編集し、暗号化通信（HTTPS: 443番ポート）を強制。通信の盗聴リスクを排除。

## 🚇 3. 運用・アクセス検証方法

NAT環境下に隠された仮想マシンに対し、安全にブラウザ接続テストを行うため、**SSHローカルポートフォワーディング（トンネリング）** を採用しました。

```text
[手元PCブラウザ] ➔ (https://localhost:8443)
       │
       ▼ 【SSH暗号化トンネル】
[物理ホスト (踏み台)] ➔ [仮想マシン (Nginx:443)]
